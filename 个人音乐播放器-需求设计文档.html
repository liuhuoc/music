需求背景与目标

背景

当前主流音乐平台（网易云音乐、QQ音乐等）功能繁杂、广告众多，且需要登录注册才能使用完整功能。对于只想安静听歌的个人用户而言，这些平台存在大量干扰。本产品定位为一款纯个人使用的本地音乐播放器，聚合多个音乐源的搜索结果，提供简洁、无广告、无需登录的听歌体验。

核心定位：去掉所有社交属性（用户头像、用户名、个人主页入口），只保留音乐搜索、播放、收藏、下载等核心功能，打造极致简洁的个人听歌工具。

产品目标

  • 聚合搜索 - 支持从多个音乐源搜索歌曲，用户可自由切换音源
  • 沉浸播放 - 全屏歌词同步显示，支持多种播放模式
  • 离线下载 - 支持歌曲下载到本地，无网络也可播放
  • 零门槛使用 - 无需注册登录，打开即可使用
  • 简洁无干扰 - 无广告、无社交推送、无多余功能

功能详情

热门搜索

  • 展示 Top 10 热门搜索词，排名 1-3 带有热度标签："超级热"（红色火焰 x3）、"很热"（橙色火焰 x2）、"热门"（蓝色火焰 x1）
  • 排名徽章颜色编码：第 1 名红色渐变、第 2 名橙色渐变、第 3 名蓝色渐变、第 4-5 名紫色/青色、第 6-10 名灰色
  • 点击任意热门搜索词，自动填入搜索框并执行搜索

搜索历史

  • 记录用户最近 20 条搜索关键词，按时间倒序排列
  • 每条历史记录右侧有删除按钮，支持单条删除
  • 顶部提供"清空历史"按钮，点击后二次确认后清空全部
  • 点击历史记录词，自动填入搜索框并执行搜索

搜索执行

  • 输入关键词后点击"搜索"按钮或键盘搜索键执行搜索
  • 搜索结果页展示匹配的歌曲列表（详见第 6 节）
  • 搜索关键词自动保存到搜索历史
  • 空关键词时提示"请输入搜索内容"
  • 无结果时显示空状态页面，提示"未找到相关歌曲，换个关键词试试"

迷你播放器

结构说明

迷你播放器是全局底部常驻组件，在除全屏播放器外的所有页面中始终显示。当有歌曲正在播放或暂停时展示，无播放内容时隐藏。

位置元素说明
最左侧专辑封面圆形缩略图，带装饰性边框
左侧歌手名当前播放歌曲的歌手名，文字过长时跑马灯滚动
中间歌曲信息歌曲名 - 歌手名（粗体显示）
右侧播放/暂停按钮圆形按钮，显示播放或暂停图标
最右侧播放队列按钮音符+列表图标，点击打开播放队列页面

交互规则

  • 点击播放/暂停按钮 - 切换当前歌曲的播放/暂停状态
  • 点击播放队列按钮 - 打开播放队列页面
  • 点击歌曲信息区域 - 展开进入全屏播放器
  • 上滑手势 - 从迷你播放器区域向上滑动，展开进入全屏播放器
  • 迷你播放器位于页面最底部，不影响页面内容滚动（内容区域底部留出播放器高度的安全距离）
  • 歌曲切换时，封面和信息平滑过渡更新

次操作栏收藏心形图标，空心/实心切换
次操作栏评论气泡图标，点击打开评论面板
次操作栏定时计时器图标，点击打开定时暂停面板
次操作栏更多三点图标，点击弹出更多操作菜单

功能详情

背景效果

  • 背景使用当前歌曲专辑封面的高斯模糊放大版，营造沉浸氛围
  • 叠加半透明深色遮罩确保歌词和控件的文字可读性
  • 背景可加入微弱的浮动光点粒子动画效果
  • 歌曲切换时背景平滑过渡（交叉淡入淡出）

歌词同步

  • 歌词按时间轴逐行滚动，当前播放行居中显示
  • 当前行使用薄荷绿高亮，字号稍大，其余行为白色半透明
  • 用户可手动上下滑动浏览歌词，松手后自动回到当前播放行
  • 点击某行歌词可跳转到对应时间点播放
  • 无歌词时显示"暂无歌词"提示

进度条

  • 显示当前播放时间（左侧）和总时长（右侧），格式 mm:ss
  • 点击进度条任意位置可跳转到对应时间点
  • 长按拖拽圆形手柄可精确调节播放位置
  • 拖拽时在进度条上方显示时间气泡

播放控制

  • 播放/暂停 - 切换播放状态，图标同步更新
  • 上一首 - 播放队列中的上一首歌曲；当前歌曲播放 3 秒内点击则重头播放当前歌曲
  • 下一首 - 播放队列中的下一首歌曲
  • 循环模式 - 三种模式循环切换：列表循环 → 单曲循环 → 随机播放，图标和文字提示随之变化

手势操作

  • 下滑手势 - 从屏幕顶部向下滑动，收起回迷你播放器
  • 左右滑动 - 在歌词区域左右滑动可切换上一首/下一首

Toast 通知

  • 收藏/取消收藏操作后，在页面顶部显示薄荷绿圆角 Toast 通知（如"收藏成功"/"取消收藏成功"），2 秒后自动消失
  • Toast 不遮挡歌词区域的主要阅读位置

  • 踩 - 点击踩图标对评论点踩，与点赞互斥
  • 回复 - 点击"回复"按钮，输入框聚焦并 @ 该用户
  • 发送评论 - 在输入框中输入内容后点击发送按钮提交评论
  • 刷新 - 点击刷新图标重新加载评论列表
  • 评论数据来自对应音源平台的公开评论接口
  • 无评论时显示空状态："暂无评论，来说点什么吧"

收藏功能

功能说明

用户可以将喜欢的歌曲加入收藏列表，方便快速找到和播放。收藏数据存储在本地，无需登录同步。

功能详情

收藏操作

  • 在歌曲操作菜单中点击"喜欢"按钮收藏歌曲，图标变为实心心形
  • 在全屏播放器底部操作栏点击心形图标收藏歌曲
  • 再次点击已收藏的歌曲取消收藏，图标恢复空心
  • 操作后显示 Toast 反馈："收藏成功"或"取消收藏成功"

收藏列表

  • 在首页提供"我的收藏"入口（可放在搜索栏下方或作为独立标签页）
  • 收藏列表按收藏时间倒序排列，显示歌曲封面、名称、歌手和音源标签
  • 支持点击播放、长按拖拽排序、左滑删除
  • 支持批量管理模式：长按进入多选，可批量取消收藏
  • 收藏列表为空时显示空状态："还没有收藏歌曲，去发现喜欢的音乐吧"

下载管理

功能说明

支持将在线歌曲下载到本地存储，在无网络环境下也能播放已下载的歌曲。

功能详情

下载操作

  • 在歌曲操作菜单中点击"下载"按钮开始下载
  • 下载过程中歌曲卡片显示下载进度条

  • 下载列表显示所有已下载歌曲，包含文件大小信息
  • 支持左滑删除已下载的歌曲（仅删除本地文件，不影响收藏状态）
  • 显示已下载歌曲的总占用存储空间
  • 无网络时，已下载歌曲可正常播放，未下载歌曲显示"需要网络连接"提示

技术架构

架构概览

技术选型建议

层推荐方案说明
跨平台框架Flutter / React Native一套代码同时支持 Android 和 iOS
音频播放just_audio / ExoPlayer支持流媒体和本地文件播放，支持后台播放
本地存储SQLite / Hive存储收藏、下载记录、搜索历史、播放队列
网络请求Dio / Retrofit搜索 API 请求、歌曲信息获取、评论获取
状态管理Provider / Riverpod / Bloc管理播放状态、UI 状态、数据流
歌词解析自定义 LRC 解析器解析标准 LRC 格式歌词文件，实现时间轴同步
后台播放audio_service / WorkManager支持锁屏播放控制、通知栏控制
下载管理flutter_downloader / DownloadManager支持断点续传、进度回调

音乐源 API 说明

注意：本产品需要对接第三方音乐搜索 API。具体 API 接口和协议不在本文档范围内，开发时需根据实际可用的音乐源进行对接。建议使用开源的音乐搜索聚合 API 项目。
  • 搜索接口：输入关键词，返回匹配的歌曲列表（含歌曲名、歌手、封面 URL、音频 URL）
  • 歌词接口：根据歌曲 ID 获取 LRC 格式歌词
  • 排行榜接口：获取热歌榜、飙升榜、新歌榜数据
  • 评论接口：根据歌曲 ID 获取评论列表
  • 热门搜索接口：获取当前热门搜索关键词列表
  • 音频流：支持在线流式播放，支持 Range 请求实现进度跳转

性能要求

指标要求
App 冷启动时间不超过 2 秒
搜索响应时间不超过 3 秒（含网络请求）
歌曲播放缓冲点击后 1 秒内开始播放
页面切换动画流畅，帧率不低于 60fps
歌词同步精度误差不超过 100ms
内存占用正常运行不超过 150MB

兼容性要求

平台最低版本
AndroidAndroid 7.0 (API 24) 及以上
iOSiOS 13.0 及以上

离线与存储

  • 已下载歌曲支持完全离线播放，包括歌词显示
  • 收藏列表、搜索历史、播放队列、播放模式等设置在离线时保留
  • 下载文件存储在 App 私有目录，卸载 App 时自动清理
  • 支持用户手动清理下载缓存

后台播放

  • App 退到后台后继续播放音乐
  • 系统通知栏显示播放控制（播放/暂停、上一首、下一首）
  • 锁屏界面显示播放控制和歌曲信息
  • 支持蓝牙耳机/车载系统的媒体控制按钮

场景处理方式
